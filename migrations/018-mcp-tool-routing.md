# Migration 018 — MCP Tool Routing Retrofit

<!-- migration-id: 018-mcp-tool-routing -->

> Retrofit existing bootstrapped projects with MCP action→tool routing: creates `.claude/rules/mcp-tool-routing.md` conditionally on `.mcp.json` presence, patches all `proj-*` agent STEP 0 force-read lists + override clauses, rewrites code-writer/test-writer/code-reviewer Section 2 Pre-Work + Section 6 Plugin/LSP/MCP to MCP-first precedence, fetches refreshed `techniques/agent-design.md`.

Note: `requires_mcp_json: false` intentionally — this migration runs in all projects. ## Problem

Bootstrap versions prior to 018 ship an MCP routing stack with three structural failures that prevent MCP tools from ever taking effect in sub-agents, even when `.mcp.json` is correctly configured and `migration 013` has stripped the `tools:` whitelist:

1. **Body-prose Pre-Work mandate outranks any MCP hint.** Agent bodies generated by `modules/05` + `modules/07` carry a Section 2 ("Before Writing") block labeled `MANDATORY` that lists `Read target file` / `2-3 similar files` / `Grep for patterns` as steps 1-2. That imperative runs on every task. Any MCP preference stated elsewhere in the body is subordinate to a step labeled MANDATORY. Models obey the MANDATORY block and never route through MCP tools.

2. **STEP 0 force-read list cites the wrong rule file.** All `proj-*` agents force-read `.claude/rules/mcp-routing.md`. That file describes *propagation* (whether agents inherit MCP tools via `tools:` frontmatter) — it has no action→tool table. The override clause `If mcp-routing.md is loaded, it OVERRIDES any Grep / Glob / Read-first examples later in this file. Route through MCP tools per that rule before falling back to text search.` points the model at a file that has no routing rules. The override is semantically inert. The rule file that *does* contain the action→tool table (`mcp-tool-routing.md`) did not exist in the bootstrap repo prior to this migration.

3. **External setup prompt drift.** A separate prompt at `\\truenas-pioneer\media\software\claude-mcp-setup-prompt.md` was being used as an alternate retrofit path and re-introduced bugs that migration 013 fixed: it re-added wildcard `tools:` entries to agent frontmatter (silently ignored by Claude Code, strips MCP access) and added `mcp__*` globs to skill `allowed-tools:` (which `.claude/rules/mcp-routing.md` explicitly says is always wrong — skill tool lists do not cascade to dispatched agents). Projects that ran that prompt are in a worse state than projects that just ran the bootstrap.

Migration 013 was the *propagation* layer fix: it stripped `tools:` from write agents so they inherit parent MCP tools. Migration 014 extended that to `proj-plan-writer`. Migration 018 is the *body-prose* layer fix: it creates the action→tool routing table, wires it into the STEP 0 force-read list of every agent, rewrites the Section 2 Pre-Work to put MCP tools before Read/Grep, and replaces the Section 6 "prefer serena" stub with an explicit precedence rule. Together, 013 + 014 + 018 make MCP tools actually reachable from sub-agents and make sub-agents actually use them.

## Changes

1. Conditionally creates `.claude/rules/mcp-tool-routing.md` (action→tool table) when `.mcp.json` declares `serena` or `codebase-memory-mcp`.
2. Conditionally inserts `@import .claude/rules/mcp-tool-routing.md` into `CLAUDE.md` (immediately after the existing `mcp-routing.md` import).
3. Patches every `.claude/agents/proj-*.md` STEP 0 force-read list to add a bullet for `mcp-tool-routing.md` immediately after the existing `mcp-routing.md` bullet (glob — covers sub-specialists created via `/evolve-agents`).
4. Rewrites the single-file override clause in every `.claude/agents/proj-*.md` to the two-file form that names both `mcp-routing.md` (propagation) and `mcp-tool-routing.md` (action table).
5. For `proj-code-writer-*.md`, `proj-test-writer-*.md`, and `proj-code-reviewer.md`: rewrites Section 2 Pre-Work to put "use MCP tools per routing table" as step 1, shifting existing steps down.
6. For the same subset: rewrites Section 6 Plugin/LSP/MCP from the legacy "prefer serena" stub to an explicit precedence rule (`MCP → LSP → Read/Grep/Glob` when `mcp-tool-routing.md` loaded; `Read/Grep/Glob` primary otherwise).
7. Fetches a refreshed copy of `techniques/agent-design.md` from the bootstrap repo into `.claude/references/techniques/agent-design.md` (the client-project layout, not the bootstrap-repo root layout).
8. Advances `.claude/bootstrap-state.json` → `last_migration: "018"` + appends `"018"` to `applied[]`.

Idempotency: every step detects the already-applied state via grep-for-marker and skips with a `SKIP:` log line. Running the migration twice is safe — the second run prints `SKIP` for every step and exits 0.

### Step 1 — Detect MCP servers from `.mcp.json`

Write detection results to `/tmp/mig018-state` so subsequent bash code blocks (each a fresh shell) can source the state instead of re-parsing `.mcp.json`. Primary parser: `python3`. Fallback parsers: `node`, `jq`. If `.mcp.json` is absent, both flags end up empty.

```bash
#!/usr/bin/env bash
set -euo pipefail

STATE_FILE="/tmp/mig018-state"
HAS_SERENA=""
HAS_CMM=""
HAS_ANY_MCP=""

if [[ -f ".mcp.json" ]]; then
  if command -v python3 >/dev/null 2>&1; then
    HAS_SERENA=$(python3 -c "import json; d=json.load(open('.mcp.json')); print('yes' if 'serena' in d.get('mcpServers',{}) else '')" 2>/dev/null || printf "")
    HAS_CMM=$(python3 -c "import json; d=json.load(open('.mcp.json')); print('yes' if 'codebase-memory-mcp' in d.get('mcpServers',{}) else '')" 2>/dev/null || printf "")
  elif command -v node >/dev/null 2>&1; then
    HAS_SERENA=$(node -e "const d=require('./.mcp.json'); console.log((d.mcpServers||{})['serena'] ? 'yes' : '')" 2>/dev/null || printf "")
    HAS_CMM=$(node -e "const d=require('./.mcp.json'); console.log((d.mcpServers||{})['codebase-memory-mcp'] ? 'yes' : '')" 2>/dev/null || printf "")
  elif command -v jq >/dev/null 2>&1; then
    HAS_SERENA=$(jq -r '.mcpServers | has("serena") | if . then "yes" else "" end' .mcp.json 2>/dev/null || printf "")
    HAS_CMM=$(jq -r '.mcpServers | has("codebase-memory-mcp") | if . then "yes" else "" end' .mcp.json 2>/dev/null || printf "")
  else
    printf "WARN: no JSON parser (python3/node/jq) available — treating .mcp.json as empty\n"
  fi
fi

if [[ -n "$HAS_SERENA" || -n "$HAS_CMM" ]]; then
  HAS_ANY_MCP="yes"
fi

{
  printf 'HAS_SERENA=%q\n' "$HAS_SERENA"
  printf 'HAS_CMM=%q\n' "$HAS_CMM"
  printf 'HAS_ANY_MCP=%q\n' "$HAS_ANY_MCP"
} > "$STATE_FILE"

printf "Detection: HAS_SERENA=%s HAS_CMM=%s HAS_ANY_MCP=%s (state written to %s)\n" \
  "${HAS_SERENA:-no}" "${HAS_CMM:-no}" "${HAS_ANY_MCP:-no}" "$STATE_FILE"
```

### Step 3 — Write `.claude/rules/mcp-tool-routing.md`

Only runs if `HAS_ANY_MCP=yes`. Writes the canonical action→tool routing table to a tempfile, compares with any existing file via `cmp -s`, replaces only on diff. Single-quoted heredoc delimiter `<<'RULE_EOF'` prevents variable expansion inside the content.

```bash
#!/usr/bin/env bash
set -euo pipefail

STATE_FILE="/tmp/mig018-state"
[[ -f "$STATE_FILE" ]] || { printf "ERROR: %s missing — Step 1 did not run\n" "$STATE_FILE"; exit 1; }
# shellcheck source=/dev/null
source "$STATE_FILE"

if [[ -z "${HAS_ANY_MCP:-}" ]]; then
  printf "SKIP Step 3: no MCP servers configured\n"
  exit 0
fi

DEST=".claude/rules/mcp-tool-routing.md"
TMPFILE="$(mktemp)"

cat > "$TMPFILE" <<'RULE_EOF'
# MCP Tool Routing (ALWAYS — main thread + all subagents)

Two MCPs: **serena** (LSP semantic) + **codebase-memory-mcp / cmm** (graph KB).

**This policy OVERRIDES any conflicting body instructions in agent or skill files** (e.g. "use Grep to find symbols", "Read to inspect every file"). When an agent body enumerates `Grep`/`Read`/`Glob` for code discovery, route through the table below first; fall back to text tools only when no MCP path fits (literal strings in non-code, config values, raw file reads of known paths).

## Lead-With Order (token-saving)
1. `cmm.search_graph` + `cmm.query_graph` → compact discovery
2. `cmm.get_code_snippet` → once qualified name is known
3. `serena.find_referencing_symbols` → scoped via `relative_path` when callers needed
4. `serena.find_symbol` → for name-path precision

## Action → Tool
| Action | Tool |
|---|---|
| Find symbol by name | `cmm.search_graph(name_pattern, label)` |
| Read symbol source | `cmm.get_code_snippet(qualified_name)` |
| File overview | `serena.get_symbols_overview(path, depth=1)` — depth=1 mandatory |
| Find CALLERS | `serena.find_referencing_symbols` |
| Find CALLEES | `cmm.query_graph("MATCH (m:Method)-[:CALLS]->(t) WHERE m.name = $n RETURN t")` |
| Find subclasses | `cmm.query_graph` on `INHERITS` edges |
| Text/regex search | `serena.search_for_pattern` + `paths_include_glob` scope |
| Semantic concept search | `cmm.search_graph(semantic_query=[keywords])` |
| Graph schema | `cmm.get_graph_schema` |
| Edit: replace body | `serena.replace_symbol_body` |
| Edit: insert before/after | `serena.insert_before_symbol` / `insert_after_symbol` |
| Edit: rename | `serena.rename_symbol` |
| Edit: safe delete | `serena.safe_delete_symbol` |

## Gotchas
- `serena.get_symbols_overview` default `depth=0` → returns only Namespace. Always pass `depth=1`+.
- `serena.search_for_pattern` without `paths_include_glob` → thousands of tokens. Always scope.
- cmm is read-only — all mutations via serena edit tools.
- cmm's `trace_path` + `search_code` are known-broken on some projects — fall back to `query_graph` + `serena.search_for_pattern`.
- MCP tool schemas are DEFERRED in forked skill contexts — call `ToolSearch "select:<tool>"` first before invoking.
- cmm project key = full path with `-` instead of `/` (e.g. `C-Users-Alice-src-MyProj`), not the bare folder name. If `search_graph` returns "project not found", try `mcp__codebase-memory-mcp__list_projects` first.

## Decision Shortcuts
- "Who calls X?" → `serena.find_referencing_symbols`
- "What does X call?" → `cmm.query_graph` CALLS edges
- "Show me X's code" → `cmm.get_code_snippet`
- "Find classes like Y" → `cmm.search_graph(name_pattern, label="Class")`
- "Grep literal" → `serena.search_for_pattern` + glob scope
- "Rename / edit symbol" → serena edit tools only
RULE_EOF

if [[ -f "$DEST" ]] && cmp -s "$TMPFILE" "$DEST"; then
  printf "SKIP Step 3: %s already up to date\n" "$DEST"
  rm -f "$TMPFILE"
else
  mv "$TMPFILE" "$DEST"
  printf "WROTE: %s\n" "$DEST"
fi
```

### Step 5 — Patch every `.claude/agents/proj-*.md` STEP 0 block

Glob-loops over every `proj-*.md` agent file (includes sub-specialists created by `/evolve-agents`). Skips files under `references/` subdirectories and the `agent-index.yaml` file. Uses `python3` with regex replacement for the two patches — `sed` escaping for backticks + forward slashes + pipe characters in the override clause is fragile, python is safer.

Two patches per file, both idempotent:

- **Patch 5a (force-read bullet)**: after the `mcp-routing.md` force-read bullet, insert a new bullet for `mcp-tool-routing.md`. Skip if bullet already present.
- **Patch 5b (override clause)**: replace the single-file override clause with the two-file form. Skip if new form already present.

```bash
#!/usr/bin/env bash
set -euo pipefail

STATE_FILE="/tmp/mig018-state"
[[ -f "$STATE_FILE" ]] || { printf "ERROR: %s missing — Step 1 did not run\n" "$STATE_FILE"; exit 1; }
# shellcheck source=/dev/null
source "$STATE_FILE"

shopt -s nullglob

patched_count=0
skipped_count=0

for agent in .claude/agents/proj-*.md; do
  # Skip nested reference files
  case "$agent" in
    *"/references/"*) continue ;;
  esac

  if [[ ! -f "$agent" ]]; then
    continue
  fi

  AGENT_FILE="$agent" python3 - <<'PY'
import os, re, sys

path = os.environ['AGENT_FILE']
with open(path, 'r', encoding='utf-8') as f:
    content = f.read()

original = content
changed = False
status = []

# ----- Patch 5a: force-read bullet insertion -----
new_bullet = "- `.claude/rules/mcp-tool-routing.md` (if present — authoritative action→tool routing; overrides any Grep/Glob/Read-first examples later in this file)"

if 'mcp-tool-routing.md' in content:
    status.append("5a=SKIP(already has bullet)")
else:
    # Match the existing mcp-routing.md bullet line (captures any leading indent)
    bullet_pat = re.compile(
        r'^([ \t]*-[ \t]+`\.claude/rules/mcp-routing\.md`[^\n]*)$',
        re.MULTILINE,
    )
    m = bullet_pat.search(content)
    if m:
        indent_match = re.match(r'^([ \t]*)', m.group(1))
        indent = indent_match.group(1) if indent_match else ''
        replacement = m.group(1) + '\n' + indent + new_bullet
        content = bullet_pat.sub(
            lambda mm, r=replacement: r,
            content,
            count=1,
        )
        changed = True
        status.append("5a=PATCHED")
    else:
        status.append("5a=SKIP(no mcp-routing.md bullet found — agent has no STEP 0 block)")

# ----- Patch 5b: override clause rewrite -----
new_clause = (
    "If `mcp-routing.md` is loaded, follow its propagation rules (tools:/allowed-tools: config). "
    "If `mcp-tool-routing.md` is loaded, it OVERRIDES any `Grep` / `Glob` / `Read`-first examples "
    "later in this file — route through MCP tools per that rule's action→tool table before falling "
    "back to text search."
)

if "follow its propagation rules" in content and "action→tool table" in content:
    status.append("5b=SKIP(already rewritten)")
else:
    # Match the legacy single-file override clause. Use a permissive regex that
    # tolerates minor whitespace/newline differences inside the sentence.
    old_pat = re.compile(
        r"If `mcp-routing\.md` is loaded, it OVERRIDES any `Grep` / `Glob` / `Read`-first "
        r"examples later in this file\.\s+Route through MCP tools per that rule before "
        r"falling back to text search\.",
        re.DOTALL,
    )
    if old_pat.search(content):
        content = old_pat.sub(new_clause, content, count=1)
        changed = True
        status.append("5b=PATCHED")
    else:
        status.append("5b=SKIP(no legacy override clause found)")

if changed and content != original:
    with open(path, 'w', encoding='utf-8') as f:
        f.write(content)
    print(f"PATCHED {path}: {', '.join(status)}")
else:
    print(f"SKIP    {path}: {', '.join(status)}")
PY

  # Accounting — check whether the file now contains the new markers to increment counters.
  if grep -q "mcp-tool-routing.md" "$agent" && grep -q "action→tool table" "$agent"; then
    patched_count=$((patched_count + 1))
  else
    skipped_count=$((skipped_count + 1))
  fi
done

shopt -u nullglob

printf "Step 5 summary: %d file(s) now have both markers; %d file(s) still missing at least one marker (likely no STEP 0 block)\n" \
  "$patched_count" "$skipped_count"
```

### Step 6 — Rewrite Section 2 Pre-Work + Section 6 Plugin/LSP/MCP for code-writer / test-writer / code-reviewer

Narrower glob than Step 5 — only the agents that actually implement code-writing or code-review workflows. Two regex rewrites per file, both idempotent:

- **Patch 6a (Section 2 Pre-Work)**: prepend a new step 1 (`If mcp-tool-routing.md loaded: use MCP tools per routing table for code discovery BEFORE Grep/Read`) to the numbered Pre-Work list. Detects the legacy list by matching `Read target file` or `Read changed files` (code-reviewer variant) near a `Pre-Work` header or the numbered block header. Skips if `MCP tools per routing table` is already present.
- **Patch 6b (Section 6 Plugin/LSP/MCP)**: replace the legacy "prefer serena" / stub body under the Section 6 header with an explicit precedence block (MCP → LSP → Read/Grep/Glob, with a non-MCP fallback note). Skips if `Precedence (when` is already present.

```bash
#!/usr/bin/env bash
set -euo pipefail

STATE_FILE="/tmp/mig018-state"
[[ -f "$STATE_FILE" ]] || { printf "ERROR: %s missing — Step 1 did not run\n" "$STATE_FILE"; exit 1; }
# shellcheck source=/dev/null
source "$STATE_FILE"

shopt -s nullglob

targets=()
for agent in .claude/agents/proj-code-writer-*.md \
             .claude/agents/proj-test-writer-*.md \
             .claude/agents/proj-code-reviewer.md; do
  case "$agent" in
    *"/references/"*) continue ;;
  esac
  if [[ -f "$agent" ]]; then
    targets+=("$agent")
  fi
done

if [[ "${#targets[@]}" -eq 0 ]]; then
  printf "SKIP Step 6: no code-writer/test-writer/code-reviewer agents found\n"
  shopt -u nullglob
  exit 0
fi

for agent in "${targets[@]}"; do
  AGENT_FILE="$agent" python3 - <<'PY'
import os, re, sys

path = os.environ['AGENT_FILE']
with open(path, 'r', encoding='utf-8') as f:
    content = f.read()

original = content
changed = False
status = []

# ----- Patch 6a: Section 2 Pre-Work — prepend MCP-first step -----
mcp_first_step = "1. If `.claude/rules/mcp-tool-routing.md` loaded: use MCP tools per routing table for code discovery BEFORE Grep/Read (see that rule's Lead-With Order)"

if "MCP tools per routing table" in content:
    status.append("6a=SKIP(already MCP-first)")
else:
    # Heuristic: find a numbered list starting with "1. Read target file" or
    # "1. Read target file (if modifying)" or "1. Read changed files in full"
    # (code-reviewer variant) — that's the legacy Pre-Work list.
    # Replace its first line with the new step 1 and renumber subsequent steps.
    prework_pat = re.compile(
        r'(^[ \t]*1\.[ \t]+Read (?:target file|changed files)[^\n]*$)',
        re.MULTILINE,
    )
    m = prework_pat.search(content)
    if m:
        # Build replacement: new step 1, then the old "Read target file" line
        # shifted to step 2, then increment subsequent numbered steps in the
        # same contiguous list block.
        old_line = m.group(1)
        # Extract indent from old line
        indent_match = re.match(r'^([ \t]*)', old_line)
        indent = indent_match.group(1) if indent_match else ''
        # Compose the new step 1 line
        new_step_1 = f"{indent}{mcp_first_step}"
        # Rewrite the matched line: "1. Read target file..." -> "2. Read target file..."
        shifted_old = re.sub(r'^([ \t]*)1\.', r'\g<1>2.', old_line)
        replacement = f"{new_step_1}\n{shifted_old}"

        # Replace just this first occurrence
        start, end = m.span(1)
        prefix = content[:start]
        suffix = content[end:]

        # Now walk forward through suffix and renumber a contiguous numbered list
        # (lines matching "^<indent>N. ..." with N in 2..9) — shift N -> N+1.
        lines = suffix.split('\n')
        out_lines = []
        in_list = True
        for ln in lines:
            if in_list:
                rm = re.match(r'^([ \t]*)(\d+)\.[ \t]+(.*)$', ln)
                if rm and rm.group(1) == indent:
                    n = int(rm.group(2))
                    if 2 <= n <= 9:
                        out_lines.append(f"{indent}{n+1}. {rm.group(3)}")
                        continue
                    else:
                        in_list = False
                else:
                    # Allow a single blank line inside the list block
                    if ln.strip() == '' and out_lines and out_lines[-1].strip() != '':
                        out_lines.append(ln)
                        continue
                    in_list = False
            out_lines.append(ln)
        suffix = '\n'.join(out_lines)

        content = prefix + replacement + suffix
        changed = True
        status.append("6a=PATCHED")
    else:
        status.append("6a=SKIP(no legacy Pre-Work list found)")

# ----- Patch 6b: Section 6 Plugin/LSP/MCP — replace stub with precedence block -----
new_section_6 = (
    "## Section 6: Plugin/LSP/MCP Requirements\n"
    "Precedence (when `.claude/rules/mcp-tool-routing.md` loaded): MCP tools → LSP operations → Grep/Glob/Read\n"
    "- MCP: see `.claude/rules/mcp-tool-routing.md` action→tool table\n"
    "- LSP (if available): rename / find-refs / go-to-def\n"
    "- Read/Grep/Glob: fallback when no MCP path fits (literal strings, config values, markup, raw file reads of known paths)\n"
    "\n"
    "When MCP servers are NOT configured: Read/Grep/Glob are primary (no fallback inversion required).\n"
)

if "Precedence (when" in content:
    status.append("6b=SKIP(already rewritten)")
else:
    # Match a Section 6 header and replace the body up to the next section header
    # (## / ### of any kind) or EOF.
    sec6_pat = re.compile(
        r'(^#{2,3}[ \t]+(?:Section[ \t]+)?6[.:]?[ \t]+Plugin/LSP/MCP[^\n]*\n)'
        r'(.*?)'
        r'(?=^#{1,3}[ \t]|\Z)',
        re.DOTALL | re.MULTILINE,
    )
    m = sec6_pat.search(content)
    if m:
        header_line = m.group(1)
        body = m.group(2)
        # Only rewrite if the body contains one of the legacy markers
        legacy_markers = ["prefer serena", "prefer Serena", "LSP plugin + operations", "recommended plugins"]
        if any(marker in body for marker in legacy_markers):
            content = content[:m.start()] + new_section_6 + "\n" + content[m.end():]
            changed = True
            status.append("6b=PATCHED")
        else:
            status.append("6b=SKIP(Section 6 present but no legacy markers — leaving as-is)")
    else:
        status.append("6b=SKIP(no Section 6 header found)")

if changed and content != original:
    with open(path, 'w', encoding='utf-8') as f:
        f.write(content)
    print(f"PATCHED {path}: {', '.join(status)}")
else:
    print(f"SKIP    {path}: {', '.join(status)}")
PY
done

shopt -u nullglob
```

### Step 7 — Sync `techniques/agent-design.md` from bootstrap repo

Per `.claude/rules/general.md` ("technique update = sync step in migration"), fetch a refreshed copy of `techniques/agent-design.md` from the bootstrap repo and write it to `.claude/references/techniques/agent-design.md` (client-project layout — **not** `techniques/` at the client-project root, that would create an orphan file).

Preferred transport: `gh api` (uses GitHub auth, handles rate limits). Fallback: `curl` against `raw.githubusercontent.com`. Resolves `BOOTSTRAP_REPO` from `.claude/bootstrap-state.json` `bootstrap_repo` key; defaults to `tomasfil/claude-bootstrap`. Idempotent: fetches to a `.new` tempfile, compares with existing via `cmp -s`, replaces only on diff.

```bash
#!/usr/bin/env bash
set -euo pipefail

BOOTSTRAP_REPO="$(python3 -c "import json; print(json.load(open('.claude/bootstrap-state.json')).get('bootstrap_repo','tomasfil/claude-bootstrap'))" 2>/dev/null || printf "tomasfil/claude-bootstrap")"

DEST_DIR=".claude/references/techniques"
DEST="${DEST_DIR}/agent-design.md"
TMPFILE="${DEST}.new"

mkdir -p "$DEST_DIR"

fetched=0
if command -v gh >/dev/null 2>&1; then
  if gh api "repos/${BOOTSTRAP_REPO}/contents/techniques/agent-design.md" --jq '.content' 2>/dev/null \
      | base64 -d > "$TMPFILE" 2>/dev/null && [[ -s "$TMPFILE" ]]; then
    fetched=1
  fi
fi

if [[ "$fetched" -eq 0 ]] && command -v curl >/dev/null 2>&1; then
  if curl -sSL --fail "https://raw.githubusercontent.com/${BOOTSTRAP_REPO}/main/techniques/agent-design.md" \
       -o "$TMPFILE" 2>/dev/null && [[ -s "$TMPFILE" ]]; then
    fetched=1
  fi
fi

if [[ "$fetched" -eq 0 ]]; then
  rm -f "$TMPFILE"
  printf "WARN Step 7: could not fetch techniques/agent-design.md from %s (gh+curl both failed or unavailable) — leaving existing file untouched\n" "$BOOTSTRAP_REPO"
  exit 0
fi

if [[ -f "$DEST" ]] && cmp -s "$TMPFILE" "$DEST"; then
  printf "SKIP Step 7: %s already up to date\n" "$DEST"
  rm -f "$TMPFILE"
else
  mv "$TMPFILE" "$DEST"
  printf "UPDATED: %s (from %s)\n" "$DEST" "$BOOTSTRAP_REPO"
fi
```
