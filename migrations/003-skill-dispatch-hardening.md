## Problem

Migration 001 renamed agents to `proj-*` but its Step 5 sed only caught four narrow patterns: backtick-wrapped `` `name` ``, `subagent_type="name"` / `subagent_type: "name"`, and line-start `Agent: name`. It missed the dominant prose forms used throughout skill files generated by v6 pre-release module 06:

1. **Bold dispatch markers** — `**Dispatch quick-check**`, `**Dispatch debugger**`, `**Dispatch plan-writer agent**`, `Dispatch **reflector** agent`, etc.
2. **Skill description frontmatter** — `Dispatches quick-check for triage, then debugger...`, `Dispatches code-reviewer agent`, `Dispatches both verifier and consistency-checker agents`.
3. **Prose mentions** — `If verifier or consistency-checker agent unavailable`, `Cluster by agent tag (code-reviewer, verifier, etc.)`, `for each code-writer-*`.
4. **Capability tables** — `code-write/references/capability-index.md` listing `code-writer-csharp`, `test-writer-csharp`, `project-code-reviewer`.

Observed symptom: user invoked `/write-plan` skill in a v6 project. Skill's procedure said `**Dispatch plan-writer agent** w/:`. Main agent either ignored the weak prose directive, misrouted to built-in `Explore` (semantic collision), or ran inline. `proj-plan-writer` never fired.

Second root cause: **skills never carried the AGENT_DISPATCH_POLICY_BLOCK**. Module 06 defines this block (forbid built-in Explore/general-purpose, stop if custom agent missing) but only post-rename skills inject it. Pre-rename bootstraps and partial 001 migrations produced skills without the policy — main agent has no strict directive to prefer custom `proj-*` over built-ins.

Third gap: even with correct names, `**Dispatch proj-X**` bold prose is a weaker signal than literal `subagent_type="proj-X"`. Main agent treats prose as guidance, not instruction. The canonical form **`Dispatch agent via \`subagent_type="proj-X"\`**` is what forces explicit dispatch.

---

## Changes

1. **Pass A — Idempotent rename**: perl lookaround regex `(?<![-a-z0-9])name(?![-a-z0-9])` rewrites every bare unprefixed agent name in `.claude/skills/**/*.md` to `proj-<name>`. Safe against `proj-name`, `code-writer-csharp`, hyphenated neighbors. Handles families (`code-writer-*`, `test-writer-*`) and the `project-code-reviewer` → `proj-code-reviewer` prefix drop.
2. **Pass B — Policy block injection**: for every skill file whose frontmatter `allowed-tools` includes `Agent`, inject the `Agent Dispatch Policy` block after the `# /skill-name` title if missing. Idempotent — skips files that already contain the string.
3. **Pass C — Canonical dispatch form**: rewrite `**Dispatch proj-<name>**` and `Dispatch **proj-<name>** agent` to the canonical literal-dispatch form `**Dispatch agent via \`subagent_type="proj-<name>"\`**`.
4. **State update** — `last_migration` → "003".

Out of scope: re-fetching skills wholesale (would clobber local edits), touching `context: fork` frontmatter (kept per module 06 note about forward compat), editing `code-write/references/capability-index.md` free-form tables beyond the rename pass.

### Step 2 — Pass A: idempotent rename to proj-* prefix

```bash
# Order matters:
#   1. project-code-reviewer → proj-code-reviewer (drops redundant 'project-')
#   2. Specific family patterns (code-writer-*, test-writer-*) BEFORE bare names
#   3. Core 12 agents
#   4. Bare test-writer / code-writer with no suffix
#
# Idempotency: (?<![-a-z0-9]) lookbehind + (?![-a-z0-9]) lookahead ensures
# we never match inside an existing proj-X or code-writer-foo compound.

for f in "${TARGETS[@]}"; do
  # 1. project-code-reviewer (drops 'project-' prefix entirely)
  perl -i -pe 's/(?<![-a-z0-9])project-code-reviewer(?![-a-z0-9])/proj-code-reviewer/g' "$f"

  # 2. Family specialists — match code-writer-<lang>, test-writer-<lang>
  perl -i -pe 's/(?<![-a-z0-9])code-writer-([a-z0-9]+)(?![-a-z0-9])/proj-code-writer-$1/g' "$f"
  perl -i -pe 's/(?<![-a-z0-9])test-writer-([a-z0-9]+)(?![-a-z0-9])/proj-test-writer-$1/g' "$f"

  # 3. Core 12 — all unprefixed canonical agent names
  perl -i -pe 's/(?<![-a-z0-9])(researcher|quick-check|debugger|verifier|reflector|consistency-checker|tdd-runner|plan-writer|code-reviewer)(?![-a-z0-9])/proj-$1/g' "$f"

  # 4. Bare test-writer / code-writer-markdown / code-writer-bash
  #    (the family patterns in step 2 would have caught code-writer-markdown,
  #    but be explicit since v6 sometimes uses bare 'test-writer' in prose)
  perl -i -pe 's/(?<![-a-z0-9])test-writer(?![-a-z0-9])/proj-test-writer/g' "$f"
done

echo "Pass A complete: proj-* rename applied to ${#TARGETS[@]} files"
```

### Step 3 — Pass B: inject AGENT_DISPATCH_POLICY_BLOCK

```bash
python3 <<'PY'
import os, re, glob

POLICY_BLOCK = """**Agent Dispatch Policy**: Use `subagent_type="proj-<name>"` explicitly.
NEVER substitute built-in `Explore` / `general-purpose` / plugin agents.
If a custom agent is missing → STOP and inform user. See `techniques/agent-design.md § Agent Dispatch Policy`.
"""

targets = sorted(set(
    glob.glob('.claude/skills/**/*.md', recursive=True) +
    glob.glob('.claude/skills/*.md')
))

injected = 0
skipped_no_agent = 0
skipped_already = 0

for path in targets:
    with open(path, encoding='utf-8') as f:
        content = f.read()

    # Idempotency — skip if block already present
    if 'Agent Dispatch Policy' in content:
        skipped_already += 1
        continue

    # Parse frontmatter
    fm_match = re.match(r'^(---\n.*?\n---\n)(.*)$', content, re.DOTALL)
    if not fm_match:
        # No frontmatter — not a canonical skill file
        skipped_no_agent += 1
        continue

    frontmatter, body = fm_match.group(1), fm_match.group(2)

    # Only inject into skills that dispatch (have Agent in allowed-tools)
    at_match = re.search(r'allowed-tools:\s*([^\n]+)', frontmatter)
    if not at_match or 'Agent' not in at_match.group(1):
        skipped_no_agent += 1
        continue

    # Find first # title line in body
    lines = body.split('\n')
    title_idx = None
    for i, line in enumerate(lines):
        if re.match(r'^#\s+', line):
            title_idx = i
            break

    if title_idx is None:
        skipped_no_agent += 1
        continue

    # Find end of title block: first blank line AFTER title content
    # Skip the title line, then any non-blank description lines, then insert at first blank
    insert_idx = title_idx + 1
    # Skip blank line right after title
    while insert_idx < len(lines) and lines[insert_idx].strip() == '':
        insert_idx += 1
    # Skip description lines (until next blank or next heading)
    while insert_idx < len(lines) and lines[insert_idx].strip() != '' and not lines[insert_idx].startswith('#'):
        insert_idx += 1

    # Insert policy block + blank separator
    new_lines = lines[:insert_idx] + ['', POLICY_BLOCK.rstrip()] + lines[insert_idx:]
    new_body = '\n'.join(new_lines)

    with open(path, 'w', encoding='utf-8') as f:
        f.write(frontmatter + new_body)
    injected += 1

print(f"Pass B complete: injected={injected}, skipped_no_agent={skipped_no_agent}, already_present={skipped_already}")
PY
```

### Step 4 — Pass C: normalize bold dispatch prose to canonical form

```bash
# Rewrite weaker prose forms to the literal subagent_type= form.
#
#   **Dispatch proj-X**                → **Dispatch agent via `subagent_type="proj-X"`**
#   **Dispatch proj-X** w/             → **Dispatch agent via `subagent_type="proj-X"`** w/
#   **Dispatch proj-X agent**          → **Dispatch agent via `subagent_type="proj-X"`**
#   Dispatch **proj-X** agent          → Dispatch agent via `subagent_type="proj-X"`
#   Dispatch **proj-X** w/             → Dispatch agent via `subagent_type="proj-X"` w/
#
# Idempotency: pattern requires the WEAK form (bold around Dispatch+name OR
# bold around bare name). The canonical output uses backticks + =, which
# the pattern will not re-match.

AGENT_NAMES='proj-researcher|proj-quick-check|proj-debugger|proj-verifier|proj-reflector|proj-consistency-checker|proj-tdd-runner|proj-plan-writer|proj-code-reviewer|proj-code-writer-[a-z0-9]+|proj-test-writer(?:-[a-z0-9]+)?'

for f in "${TARGETS[@]}"; do
  # Form 1: **Dispatch <name> agent** / **Dispatch <name>**
  perl -i -pe "s/\\*\\*Dispatch (${AGENT_NAMES})(?:\\s+agent)?\\*\\*/**Dispatch agent via \`subagent_type=\"\$1\"\`**/g" "$f"

  # Form 2: Dispatch **<name>** agent / Dispatch **<name>**
  perl -i -pe "s/Dispatch \\*\\*(${AGENT_NAMES})\\*\\*(?:\\s+agent)?/Dispatch agent via \`subagent_type=\"\$1\"\`/g" "$f"
done

echo "Pass C complete: bold dispatch prose normalized in ${#TARGETS[@]} files"
```

## Rules for migration scripts

- **Idempotent** — all three passes use lookaround / presence-checks so re-running is a no-op. Pass A's perl lookaround prevents `proj-proj-*`. Pass B skips files that already contain `Agent Dispatch Policy`. Pass C's pattern matches only the weak form; canonical output won't re-match.
- **Glob agent filenames, never hardcode** — Pass A handles `code-writer-*` / `test-writer-*` families via regex capture, not hardcoded lang list.
- **Read-before-write** — every file is read by perl/python before rewrite.
- **Abort on error** — `set -euo pipefail` at top; every step has non-zero exit on failure.
